02Deeplabv3+语义分割网络

发表于2026-03-02|更新于2026-03-03|语义分割
|浏览量:

DeepLabv3+ 主要解决像素级语义分割中的两个关键矛盾：

  • 语义信息与分辨率的矛盾：下采样越多语义越强，但边界细节越差
  • 多尺度目标的矛盾：同一场景中目标尺度变化大，需要多尺度上下文

DeepLabv3+ 通过空洞卷积控制感受野，通过 ASPP 进行多尺度特征融合，并通过解码器补回细节。

核心组件

1) 空洞卷积（Atrous Convolution）

空洞卷积在卷积核元素之间插入间隔，从而在不增加参数量、不额外下采样的情况下扩大感受野。

  • dilation rate 越大，感受野越大
  • 适合在保持特征图分辨率时提取更大范围语义

在 DeepLab 系列中，空洞卷积常用于替代部分 stride=2 的下采样，配合输出步长（Output Stride, OS）控制最终特征图大小。

2) ASPP（Atrous Spatial Pyramid Pooling）

ASPP 使用多分支并行结构，在不同 dilation rate 下做空洞卷积，并融合输出特征，实现多尺度上下文建模。

典型 ASPP 分支包括：

  • 1x1 卷积分支
  • 多个 3x3 空洞卷积分支（不同 dilation rate）
  • 全局平均池化分支（全局上下文）

这些分支拼接后再经过 1x1 卷积融合，得到具备强多尺度语义表达的特征。

3) 编码器-解码器结构

DeepLabv3 是强语义编码器，DeepLabv3+ 在其基础上加入解码器，提升边界与细节：

  • 编码器输出经过 ASPP 得到高层语义特征
  • 解码器将高层特征上采样
  • 同时引入浅层低级特征（高分辨率但语义弱）进行融合
  • 经过少量卷积细化后输出分割结果

这种设计实现了“语义 + 细节”的兼顾。

结构流程

DeepLabv3+ 的典型流程如下：

  1. 输入图像经过骨干网络（Backbone）提取特征
  2. 在高层特征上应用 ASPP 获取多尺度语义
  3. 将 ASPP 输出上采样到更高分辨率
  4. 从骨干网络提取一层浅层特征（通常更高分辨率）并降维
  5. 将上采样后的高层特征与浅层特征拼接
  6. 通过解码器卷积细化后再上采样到输入大小
  7. 1x1 卷积输出每像素类别概率

Backbone 与 Output Stride

常用骨干网络包括：

  • ResNet-50 / ResNet-101（经典、稳定）
  • Xception（DeepLab 原论文常用，效果强）
  • MobileNetV2/V3（轻量化部署）

Output Stride（OS）常见选择：

  • OS=16：速度更快，精度较好
  • OS=8：特征分辨率更高，边界更好，但计算更重

工程上常先用 OS=16 训练验证，再视需求切换 OS=8 精调。
