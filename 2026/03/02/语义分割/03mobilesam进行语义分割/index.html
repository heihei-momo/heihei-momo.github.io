03Mobilesam+clip零样本图像语义分割

发表于2026-03-02|更新于2026-03-03|语义分割
|浏览量:

MobileSAM 是对 Segment Anything Model（SAM）的轻量化改造版本，目标是在资源受限的设备（手机/边缘端/CPU）上也能“像 SAM 一样”做提示驱动分割（point/box prompt）。MobileSAM 的核心做法是替换 SAM 中最重的图像编码器，并通过一种称为**解耦蒸馏（decoupled distillation）**的方法，让新编码器输出的图像特征能与原 SAM 的 mask decoder 保持兼容。

SAM 的基本范式

SAM 的定位是“可提示的分割基础模型（promptable segmentation foundation model）”。它的典型推理流程是：

  1. Image Encoder：把整张图像编码成一个高维 embedding（一次性计算，可复用）
  2. Prompt Encoder：把点/框/粗 mask 等提示编码成 prompt embedding
  3. Mask Decoder：结合图像 embedding + prompt embedding 输出分割 mask（可多次交互）

SAM 的关键能力是“同一张图像上，多次不同 prompt 的交互式分割”，因此图像编码器的速度非常重要。

MobileSAM 的核心改动

只换 Image Encoder，其它尽量不动

MobileSAM 保持与原 SAM 相同的整体 pipeline、预处理/后处理、prompt 方式与 mask decoder，仅将原本重量级的 ViT-H 图像编码器替换为更轻量的编码器。这样做带来的直接好处是：

  • 迁移成本极低，原来用 SAM 的代码几乎不需要大改
  • prompt 交互体验保持一致（同样支持点/框等提示）

用 TinyViT 替换 ViT-H

MobileSAM 在官方说明中明确给出了替换关系与量级差异：

  • 原 SAM 图像编码器：ViT-H（超大参数量）
  • MobileSAM 图像编码器：TinyViT（约 5M 级别）

这意味着 MobileSAM 的主要加速来自图像编码阶段，而 decoder 仍保持 SAM 原版结构与规模。

解耦蒸馏

直觉上，“把 ViT-H 换成轻量编码器，然后按 SAM 的方式训练”并不可靠。MobileSAM 论文指出，直接端到端训练会出现性能不理想的问题，主要原因是图像编码器与 mask decoder 的耦合优化在训练资源有限时会变得困难。

为此，MobileSAM 提出decoupled distillation（解耦蒸馏）：

  • 教师端：原 SAM 的重编码器（ViT-H）产生高质量图像 embedding
  • 学生端：轻量编码器学习去“对齐”教师 embedding
  • 目标：让学生编码器输出的 embedding 能直接喂给原 SAM 的 mask decoder，从而实现“编码器替换但整体行为保持一致”

这种思路本质上是在做“接口对齐”：只要 embedding 的分布与语义足够接近，decoder 就能继续工作。

推理流程与交互方式

MobileSAM 的推理流程与 SAM 基本一致：

  1. 输入图像 → 轻量 Image Encoder（TinyViT）→ 图像 embedding
  2. 输入 prompt（点/框）→ Prompt Encoder → prompt embedding
  3. 图像 embedding + prompt embedding → Mask Decoder → mask（可输出多候选 mask）

因此它仍然适合以下交互式任务：

  • 点一下分割一个目标
  • 框选一个区域分割前景
  • 多点提示细化分割边界
  • 标注辅助、抠图、图像编辑工作流等（更偏工程应用）

速度与模型规模

MobileSAM 论文与仓库给出了一些代表性指标（不同设备会有差异）：

  • 论文摘要中提到：MobileSAM 相比原版 SAM 小 60 倍以上，并且在 GPU 上推理可达到很快的图像编码速度，同时保持接近原 SAM 的效果。
  • 仓库给出：整条 Enc+Dec pipeline 的参数量与耗时对比，并强调主要收益来自编码器替换。

工程上理解为一句话更实用：交互式分割的瓶颈通常在编码器，MobileSAM 把这个瓶颈大幅压缩了。

全局分割

CLIP（Contrastive Language–Image Pretraining）是一类把图片和文字“对齐”到同一个向量空间的模型：它同时训练一个图像编码器和一个文本编码器，把一张图和一句话各自编码成向量，然后用对比学习让“匹配的图文对”相似度更高、“不匹配的”更低。训练好之后，CLIP 就能用相似度检索的方式做很多事。
