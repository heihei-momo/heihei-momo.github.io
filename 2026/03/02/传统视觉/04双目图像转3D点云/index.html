04双目图像转3D点云

发表于2026-03-02|更新于2026-03-03|传统视觉
|浏览量:

总览

双目稠密重建流程。程序读取左右相机的灰度图像，使用 OpenCV 的 StereoSGBM 算法计算视差图。程序根据双目几何模型将视差转换为深度，再将像素坐标反投影为三维点。程序将左图的灰度值归一化后作为点的强度信息，并用 Pangolin 在三维窗口中显示点云。

依赖与模块分工

  • OpenCV 负责读取左右图、执行 StereoSGBM 视差计算、显示视差图。
  • Eigen 用于存储点云元素，点使用 Vector4d 表示。
  • Pangolin 负责创建窗口、相机交互、绘制点云。

输入与参数约定

程序使用固定路径读取左右图像：

  • left.png：左相机灰度图
  • right.png：右相机灰度图

程序假设内参与基线已知并且与图像匹配：

  • fx, fy, cx, cy 为相机内参。
  • b 为双目基线，单位与最终深度单位一致。

点云数据结构

程序使用 Vector4d 存储单个点：

  • point[0], point[1], point[2] 为三维坐标 (X, Y, Z)
  • point[3] 为强度值 I，由左图灰度归一化得到

流程步骤

1) 读取左右灰度图

程序将左右图以灰度方式读入，确保立体匹配输入为单通道图像：

2) 配置 StereoSGBM 并计算视差图

程序创建 cv::StereoSGBM 实例并计算视差。SGBM 的输出视差以定点形式存储，OpenCV 默认使用 16 倍缩放，因此程序将其转为 float 并除以 16.0 得到真实视差值：

关键参数约定如下：

  • minDisparity = 0
  • numDisparities = 96，视差搜索范围为 [0, 96)，该值通常需要为 16 的倍数
  • blockSize = 9，匹配窗口大小

3) 过滤无效视差并恢复深度

程序对每个像素位置读取视差值并过滤无效与越界视差：

程序依据双目模型计算深度，视差越大代表深度越小，视差越小代表深度越大。

4) 像素坐标反投影为三维点

程序将像素坐标归一化后乘以深度得到三维坐标：

  • x = (u - cx) / fx
  • y = (v - cy) / fy
  • X = x * Z
  • Y = y * Z
  • Z = depth

该过程等价于针孔相机模型的反投影，程序将所有有效像素的点加入点云容器。

5) 将灰度值作为点强度并累积点云

程序从左图读取灰度并归一化到 [0, 1]，作为第四维强度值，用于渲染颜色：

程序将点压入 pointcloud：

程序保留了降采样提示，用于在算力较弱时减少点数：

  • 将 v++ 和 u++ 修改为 v+=2、u+=2 以进行 2 倍降采样。

6) 可视化视差图

程序将视差除以 96.0 映射到 [0, 1] 区间并显示。该显示用于直观观察视差分布与匹配质量：

7) 使用 Pangolin 绘制点云

程序在 Pangolin 中创建窗口与相机控制器，并使用 OpenGL 点绘制显示点云。程序将强度值 p[3] 作为灰度颜色写入 RGB 通道：

程序通过 usleep(5000) 将刷新间隔控制在约 5ms，避免占满 CPU。

输出结果与观察方式

  • 视差结果通过 OpenCV 窗口展示。
  • 点云结果通过 Pangolin 三维窗口展示。
  • 点云为稠密点集合，点的颜色为灰度强度，几何形状由深度恢复决定。
