02局部路径规划

总览

实现了一个 ROS nav_core::BaseLocalPlanner 插件形式的局部规划器 my_planner::MyPlanner。规划器接收上层给定的全局路径 global_plan_，在每个控制周期内根据机器人当前位姿选择一个临时目标点，并输出线速度与角速度 cmd_vel。规划器使用 PID 思路对横向偏差进行角速度控制，并对速度进行限幅。规划器同时读取局部代价地图（local costmap）进行禁行区域检测，当检测到前方路径点落在禁行区域时返回 false 触发重新全局规划。规划器在接近终点后进入终点姿态调整分支，完成最终角度对齐并置位 goal_reached_。

插件注册与控制参数

插件通过 PLUGINLIB_EXPORT_CLASS 注册为 nav_core::BaseLocalPlanner：

该实现使用 PID 控制转向，参数为：

  • Kp = 4.5
  • Ki = 0.0
  • Kd = 0.5

PID 状态变量包括：

  • angular_error：当前横向误差
  • last_error：上一周期误差
  • error_sum：误差累计（积分项）
  • error_diff：误差变化（微分项）
  • output：PID 输出，用作角速度

代码构建一张 CV_8UC3 图像，并根据代价值上色：

2) 对前方一段路径做禁行区探测

代码对 target_index_ 起始的后续 10 个路径点做检测。若对应栅格代价值 >= 253，函数直接返回 false，触发上层重新规划全局路径：

该策略属于“前方安全检查”。该策略以路径点落入禁行/障碍区域作为快速失败条件，避免机器人继续沿不可行路径运动。

终点阶段处理：终点姿态对齐

代码对终点采用“距离触发 + 姿态闭环”的分支处理：

  1. 将全局路径终点转换到 base_link 坐标系
  2. 当终点距离小于阈值（代码中为 0.1m）时进入姿态调整模式 pose_adjusting_ = true
  3. 在姿态调整模式下，输出速度直接基于终点相对位姿：
    • 线速度与 x 成比例
    • 角速度与 final_yaw 成比例
  4. 当 abs(final_yaw) < 0.1 时认为到达终点并置位 goal_reached_

速度限幅

代码对线速度与角速度进行硬限幅：

该限制保护底盘执行器并提升控制稳定性。
